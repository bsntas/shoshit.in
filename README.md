# शोषितका शब्दहरू — Website for Muktinath Sharma 'Shoshit'

A fast, dependency-free static website (pure HTML + CSS, no build step) presenting the
literary works of Muktinath Sharma 'Shoshit' (मुक्तिनाथ शर्मा 'शोषित'), Bijanbari, Darjeeling.

## Structure

```
shoshit-site/
├── index.html                     # Home — hero + book shelf + author intro
├── parichay.html                  # परिचय (autobiographical verse + facts)
├── css/style.css                  # All styling (design tokens documented at top)
└── books/
    ├── man-ra-ma.html             # मन र म (2006) — COMPLETE: all 56 poems
    ├── prakaran.html              # प्रकरण (2011) — book page, texts to be added
    ├── dochhaya.html              # दोछाया (2023) — book page, texts to be added
    ├── shoshitka-shabdaharu.html  # शोषितका शब्दहरू — cover page
    └── ujyalo-khojdai.html        # उज्यालो खोज्दै — cover page
```

**Content status:** `man-ra-ma.html` contains the complete text of all 56 poems with a
table of contents. The prakaran and dochhaya pages are ready-designed book pages; each
contains an HTML comment showing the exact `<article class="piece">` template to paste
essays/stories into (copy the pattern from `man-ra-ma.html` for reference — use
`class="verse"` for poems and `class="prose-flow"` for prose, with a blank line between
paragraphs).

## Deploy to GitHub Pages

1. Create a new repository on GitHub (e.g. `shoshit-site`). For a
   `username.github.io` root site, name the repo exactly `username.github.io`.
2. Push the site:
   ```bash
   cd shoshit-site
   git init
   git add .
   git commit -m "शोषितका शब्दहरू — initial site"
   git branch -M main
   git remote add origin https://github.com/USERNAME/REPO.git
   git push -u origin main
   ```
3. On GitHub: **Settings → Pages → Source: Deploy from a branch → Branch: main / (root) → Save.**
4. The site goes live at `https://USERNAME.github.io/REPO/` within a minute or two.

### Custom domain (shoshit.in)

1. In **Settings → Pages → Custom domain**, enter `shoshit.in` (GitHub creates a `CNAME` file).
2. At your DNS provider add:
   - `A` records for the apex `shoshit.in` → `185.199.108.153`, `185.199.109.153`,
     `185.199.110.153`, `185.199.111.153`
   - `CNAME` record for `www` → `USERNAME.github.io`
3. Tick **Enforce HTTPS** once the certificate is issued.

## Design notes

- Fonts: Yatra One (display), Noto Serif Devanagari (body), Mukta (UI) via Google Fonts.
- Colour tokens (paper, sindoor, tea-green, Kanchenjunga dusk, marigold gold) are defined
  as CSS variables at the top of `css/style.css`.
- The home-page "book shelf" covers are pure CSS — replace them with real cover
  photographs anytime by swapping the `.cover` divs for `<img>` tags.
- Fully responsive; reading pages have a sticky poem/story index on desktop and a
  collapsible one on mobile. Devanagari numbering is generated by CSS counters.
